# PRD v1 — tese

**Format:** Lenny 1-pager hybrid · **Type:** personal tool · **Status:** draft · **Date:** 2026-06-02

## Problem

Investing abroad directly from Brazil (IBKR, global custody) is achievable but pushes the entire
**record-keeping and tax burden** onto the investor, while no single surface reconciles the
*thesis watched* with the *portfolio held*. Spreadsheets are fragile and lossy; brokers don't
emit a Brazil-shaped record. The owner needs a local tool that (a) shows thesis × portfolio in
one view and (b) captures every IRPF/CBE/estate-tax field from the first remittance onward.

## Why now

- Owner is actively rotating BTC → global tech equities **this month** — aporte #1 is imminent,
  and the cheapest moment to start the ledger is *before* the first remittance, not at IRPF time.
- Lei 14.754/2023 made foreign financial gains an **annual** taxable event (~15%), raising the
  cost of poor records.

## Success metrics

- **NSM — Filing-readiness:** a one-command export produces a complete, contador-ready year record
  (positions, remittances, dividends, FX, 31/12 balances) with zero manual reconstruction.
- Time-to-snapshot: open the app → see thesis + portfolio reconciled in < 5s.
- Zero missed obligation events: threshold crossings (US$ 60k US-situs, US$ 1M abroad) surfaced
  the moment they happen.

## Solution sketch

A local Python (FastAPI + SQLite) app with two data planes behind clean adapters:
- **Market-data plane (A):** quotes/day-change/fundamentals for the thesis watchlist (incl. unheld).
- **Portfolio plane (B):** positions/cost-basis/P&L — manual `holdings.yaml` first, IBKR live later.
- A **ledger** records every economic event (remittance, buy, sell, dividend, withholding) in a
  Brazil-tax-shaped schema, plus year-end (31/12) balance snapshots.
- A dashboard reads all three; an **export** produces the contador-ready package.

## User stories (US-NN, stable IDs)

**P0 — must-have for v1**
- **US-01** As the owner, I see my thesis watchlist with live quote + day change, including names I
  don't yet hold, so I track the category's movement.
- **US-02** As the owner, I see my holdings with quantity, avg cost, current price, **portfolio
  weight**, and **P&L**, reconciled against the watchlist.
- **US-03** As the owner, I record a **remittance** (date, BRL amount, FX rate, IOF, SWIFT fee,
  USD credited) so the BRL cost base of my foreign capital is captured.
- **US-04** As the owner, I record a **buy/sell** (date, ticker, exchange, currency, qty, price,
  commission, FX-to-BRL) so cost basis and realized P&L are tracked per Brazilian rules.
- **US-05** As the owner, I record **dividends** received and **foreign tax withheld** (e.g. US 30%).
- **US-06** As the owner, I capture a **31/12 balance** snapshot per position (qty + price + BRL
  value) — the figure IRPF "Bens e Direitos" requires.
- **US-07** As the owner, I **export a contador-ready package** (CSV/JSON + summary) for a tax year.

**P1 — should-have**
- **US-08** As the owner, I'm **warned** when US-situs assets approach US$ 60k (estate-tax planning)
  or total foreign assets approach US$ 1M (CBE annual filing).
- **US-09** As the owner, I tag each holding by **market** (US / HK-China / KR) and **instrument**
  (stock / ETF / ADR / UCITS) so satellite exposure and US-situs totals are computable.
- **US-10** As the owner, I swap the manual holdings source for the **live IBKR feed** via config,
  with no UI change.

**P2 — later**
- **US-11** As the owner, I see a simple time series of portfolio value and category movement.
- **US-12** As the owner, I get fundamentals/news per ticker (Finnhub) for diligence.

## Release scope (v1 = Phase 1)

In: US-01..US-07 (dashboard + manual ledger + export). Out (deferred): live IBKR (US-10, Phase 2),
threshold alerts (US-08, Phase 2), time series & fundamentals (P2), multi-year analytics.

## Anti-goals (explicit)

- No recommendations, signals, scoring, or ranking. **Decision-support only.**
- No order execution / trading. Read-only.
- No auto-filing of taxes; the tool organizes the record, the contador files.
- No multi-user, auth, hosting, pricing, or marketing in v1.
- No tax *calculation* of the owner's liability beyond informational roll-ups (the contador owns
  the official numbers; the tool owns the *data*). See `compliance-tax.md` § Boundaries.
